32 Project Mission
33 -----------------
34 
35 Project mission is to provide easy to use, powerful, C/C++ embeddable web
36 server with optional CGI, SSL and Lua support.
37 CivetWeb has a MIT license so you can innovate without restrictions.
38 
39 CivetWeb can be used by developers as a library, to add web server functionality to an existing application.
40 It can also be used by end users as a stand-alone web server. It is available as single executable, no installation is required.

78 Overview
79 --------
80 
81 CivetWeb keeps the balance between functionality and
82 simplicity by a carefully selected list of features:
83 
84 - Liberal, commercial-friendly, permissive,
85  [MIT license](http://en.wikipedia.org/wiki/MIT_License)
86 - Free from copy-left licenses, like GPL, because you should innovate without
87  restrictions.
88 - Forked from [Mongoose](https://code.google.com/p/mongoose/) in 2013, before
89  it changed the licence from MIT to commercial + GPL. A lot of enchancements
90  have been added since that time, see
91  [RELEASE_NOTES.md](https://github.com/civetweb/civetweb/blob/master/RELEASE_NOTES.md).
92 - Works on Windows, Mac, Linux, UNIX, iPhone, Android, Buildroot, and many
93  other platforms.
94 - Scripting and database support (Lua scipts, Lua Server Pages, CGI + SQLite
95  database, Server side javascript).
96  This provides a ready to go, powerful web development platform in a one
97  single-click executable with **no dependencies**.
98 - Support for CGI, HTTPS (SSL/TLS), SSI, HTTP digest (MD5) authorization, Websocket,
99  WEbDAV.
100 - Optional support for authentication using client side X.509 certificates.
101 - Resumed download, URL rewrite, file blacklist, IP-based ACL, Windows service.
102 - Download speed limit based on client subnet or URI pattern.
103 - Simple and clean embedding API.
104 - The source is in single file to make things easy.
105 - Embedding examples included.
106 - HTTP client capable of sending arbitrary HTTP/HTTPS requests.
107 - Websocket client functionality available (WS/WSS).

136 Contributions
137 ---------------
138 
139 Contributions are welcome provided all contributions carry the MIT license.
140 
141 DO NOT APPLY fixes copied from Mongoose to this project to prevent GPL tainting.
142 Since 2013, CivetWeb and Mongoose are developed independently.
143 By now the code base differs, so patches cannot be safely transferred in either direction.
144

◆ addColumn()

RooAbsArg * RooVectorDataStore::addColumn ( RooAbsArg &  newVar,
Bool_t  adjustRange = kTRUE 
)
virtual

Add a new column to the data set which holds the pre-calculated values of 'newVar'.

This operation is only meaningful if 'newVar' is a derived value.

The return value points to the added element holding 'newVar's value in the data collection. The element is always the corresponding fundamental type of 'newVar' (e.g. a RooRealVar if 'newVar' is a RooFormulaVar)

Note: This function is explicitly NOT intended as a speed optimization opportunity for the user. Components of complex PDFs that can be precalculated with the dataset are automatically identified as such and will be precalculated when fitting to a dataset

By forcibly precalculating functions with non-trivial Jacobians, or functions of multiple variables occurring in the data set, using addColumn(), you may alter the outcome of the fit.

Only in cases where such a modification of fit behaviour is intentional, this function should be used.

Upon a get() call, the internal cache of 'newVar' will be loaded with the precalculated value and it's dirty flag will be cleared.
